## 🔧 ARCHITECTURE TECHNIQUE

### Machine à états

```
BROUILLON ──activate──→ ACTIF ──start──→ EN_COURS
    │                      │                │
    │                      │                ├─pause──→ EN_ATTENTE
    │                      │                │             │
    │                      │                │             └─start──→ EN_COURS
    │                      │                │
    │                      └─complete───────┴──→ TERMINE
    │                      │                │
    │                      └─cancel─────────┴──→ ANNULE
    │                                       │
    └─delete (BROUILLON only)              └──archive──→ ARCHIVE
                                                          │
                                                          └─unarchive──→ TERMINE/ACTIF
```

**Transitions validées** :
- BROUILLON : peut → ACTIF, suppression autorisée
- ACTIF : peut → EN_COURS, ANNULE
- EN_COURS : peut → EN_ATTENTE, TERMINE, ANNULE
- EN_ATTENTE : peut → EN_COURS, ANNULE
- TERMINE : peut → ARCHIVE seulement
- ANNULE : peut → ARCHIVE seulement
- ARCHIVE : peut → désarchivage (restaure TERMINE ou ACTIF)

---

### Gestion progression

**Auto-complétion** :
- Si `progressionPct = 100%` → status auto → TERMINE
- `dateClotureReelle` auto-renseignée

**Validation** :
- Progression strictement 0-100%
- Update validation dans service

---

### Isolation données

**Multi-tenant** :
- Tous les queries filtrent par `cabinetId`
- Génération référence unique par cabinet + année
- Relations cascade delete depuis Cabinet

**Sécurité** :
- Authentification requireAuth sur toutes routes
- Validation isRegularUser
- Gestion erreurs 401/403
