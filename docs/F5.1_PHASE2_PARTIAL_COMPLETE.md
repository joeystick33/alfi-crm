## 📊 FONCTIONNALITÉS IMPLÉMENTÉES

### Création Invoice

**Workflow complet** :
1. Validation client existe
2. Génération numéro unique (INV-YYYY-NNN)
3. Calcul automatique totaux depuis items
4. Création invoice + items en transaction
5. Relations projet/opportunité optionnelles
6. Statut initial : DRAFT
7. Include complet (client, conseiller, items, payments)

**Calculs automatiques** :
- `totalHT = Σ(quantity × unitPrice)`
- `totalTTC = Σ(totalHT × (1 + tva/100))`
- `avgTVA = ((totalTTC - totalHT) / totalHT) × 100`
- Arrondi 2 décimales

### Filtres Avancés

**Liste invoices avec** :
- Status (DRAFT, SENT, PAID, OVERDUE, CANCELLED)
- Client ID
- Conseiller ID
- Projet ID
- Opportunité ID
- Issue date range (from/to)
- Due date range (from/to)
- Amount range (min/max HT)
- Pagination (limit/offset)
- Sorting (5 champs + asc/desc)

### Gestion Items

**Opérations** :
- Ajout ligne avec ordre automatique
- Suppression ligne
- Recalcul automatique totaux invoice
- Protection : modification DRAFT only

### Gestion Paiements

**Workflow** :
- Enregistrement paiement (montant, méthode, référence)
- Calcul total payé (sum payments)
- Auto-passage PAID si `totalPaid >= amountTTC`
- Protection : pas sur factures CANCELLED

### Transitions Statut

**Machine à états** :
```
DRAFT ──────┐
  │         │
  │ send    │ cancel
  ↓         ↓
SENT ────→ CANCELLED
  │
  │ (dueDate < now)
  ↓
OVERDUE
  │
  │ markAsPaid / addPayment (complet)
  ↓
PAID
```

**Protections** :
- DRAFT : modifiable, supprimable
- SENT : peut → PAID, OVERDUE, CANCELLED
- PAID : immuable, non annulable
- OVERDUE : peut → PAID, CANCELLED
- CANCELLED : immuable

### Validations Métier

**canSendInvoice()** :
- Status = DRAFT
- Items > 0
- Due date définie
- Client associé

**canMarkAsPaid()** :
- Status ≠ PAID
- Status ≠ CANCELLED
- Status ≠ DRAFT (doit être SENT/OVERDUE)

**isInvoiceOverdue()** :
- Status ≠ PAID/CANCELLED
- dueDate < now
