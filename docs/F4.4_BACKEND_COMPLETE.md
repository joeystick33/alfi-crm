## 📋 ROUTES API CRÉÉES

### 1. Liste & Création

**GET /api/advisor/conseillers**
- Liste tous les conseillers du cabinet
- Filtres : role, isActive, search, pagination, sortBy
- Include stats complètes (_count)
- Retourne : ID, nom, email, role, permissions, isActive, lastLogin, stats

**POST /api/advisor/conseillers**
- Création nouveau conseiller
- Validation Zod stricte
- Vérification email unique
- Génération mot de passe temporaire (12 chars sécurisé)
- Hash bcrypt (12 rounds)
- Permission : ADMIN only
- Retourne : Conseiller créé + tempPassword

---

### 2. CRUD Individuel

**GET /api/advisor/conseillers/[id]**
- Détail conseiller avec stats
- Include _count relations
- Vérification cabinetId (tenant isolation)

**PATCH /api/advisor/conseillers/[id]**
- Mise à jour conseiller
- Validation Zod partielle
- Vérification email unique si changé
- Permission : ADMIN only
- Update fields : email, firstName, lastName, phone, avatar, role, permissions, isActive

**DELETE /api/advisor/conseillers/[id]**
- Soft delete (isActive = false)
- Protection : Impossible de supprimer soi-même
- Permission : ADMIN only
- Préserve intégrité données relations

---

### 3. Statistiques Détaillées

**GET /api/advisor/conseillers/[id]/stats**

**Statistiques clients** :
- Total clients (principal + remplaçant)
- Clients principaux
- Clients remplaçants

**Statistiques tâches** :
- Total tâches
- Par statut : TODO, IN_PROGRESS, COMPLETED
- Tâches en retard (dueDate < now)
- Taux complétion (completed / total)

**Statistiques rendez-vous** :
- Total rendez-vous
- RDV mois en cours
- RDV à venir (upcoming)

**Statistiques opportunités** :
- Total opportunités
- Opportunités actives (en cours)
- Opportunités gagnées (WON)
- Opportunités perdues (LOST)
- Taux conversion (WON / (WON + LOST))
- Valeur pipeline totale
- Valeur pipeline pondérée
- Valeur totale gagnée

---

### 4. Gestion Assignments Assistants

**GET /api/advisor/conseillers/[id]/assignments**
- Liste tous les assistants assignés à un conseiller
- Include détails assistant (nom, email, avatar, isActive)
- Retourne permissions par assignment

**POST /api/advisor/conseillers/[id]/assignments**
- Assigner un assistant à un conseiller
- Validation : Assistant doit avoir role ASSISTANT
- Vérification : Assignment unique
- Permissions granulaires par assignment
- Permission : ADMIN only

**DELETE /api/advisor/conseillers/[id]/assignments?assistantId=xxx**
- Retirer assignment assistant
- Query param : assistantId requis
- Permission : ADMIN only

## 🔒 SÉCURITÉ & VALIDATION

### Validation Zod

**CreateConseillerSchema** :
```typescript
{
  email: z.string().email().toLowerCase(),
  firstName: z.string().min(1).max(100),
  lastName: z.string().min(1).max(100),
  phone: z.string().optional().nullable(),
  role: z.enum(['ADVISOR', 'ASSISTANT', 'MANAGER']),
  permissions: z.record(z.boolean()).optional().nullable(),
  avatar: z.string().url().optional().nullable(),
}
```

**UpdateConseillerSchema** : Partial du create

**AssignmentSchema** :
```typescript
{
  assistantId: z.string().cuid(),
  permissions: z.record(z.boolean()).optional().nullable(),
}
```

- Réservé : ADMIN only

**Consultation** :

**Assignments** :
- ADMIN only pour création/suppression
- Vérification role ASSISTANT pour assignés

```typescript

```

**Taux conversion** :
```typescript
conversionRate = Math.round((opportunitiesWon / (won + lost)) * 100)
```

```typescript
weightedValue = sum(estimatedValue × (probability / 100))
```

**Rationale** :
- Préserve intégrité référentielle (clients, tâches, RDV)
- Audit trail conservé
- Réactivation possible
- isActive = false au lieu de DELETE
