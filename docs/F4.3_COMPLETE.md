## 🔧 Détails Techniques

### Drag & Drop @dnd-kit

**Configuration Sensors** :
```typescript
const sensors = useSensors(
  useSensor(PointerSensor, {
    activationConstraint: {
      distance: 8, // 8px minimum pour activer drag
    },
  })
)
```

**Flow Drag** :
1. onDragStart → setActiveCard + setDragging
2. User drag over columns → isOver highlight
3. onDragEnd → determineTargetColumn + API call
4. Optimistic update local state
5. Si erreur → rollback + toast error

**Collision Detection** :
- closestCorners : Optimal pour colonnes adjacentes
- Active card ID : String pour flexibilité
- Over ID : Peut être colonne ou carte

### API Calls

**Tâches** :
```typescript
// Change status
PATCH /api/advisor/taches/[id]
Body: { status: 'IN_PROGRESS' }

// Complete
PATCH /api/advisor/taches/[id]
Body: { status: 'COMPLETED', completedAt: ISO_DATE }

// Delete
DELETE /api/advisor/taches/[id]
```

**Opportunités** :
```typescript
// Change status
PATCH /api/advisor/opportunites/[id]
Body: { status: 'QUALIFIED' }

// Convert to projet
POST /api/advisor/opportunites/[id]/convert
Body: { projectData: {} }

// Delete
DELETE /api/advisor/opportunites/[id]
```

### Performance

**Optimisations** :
- useMemo pour colonnes Kanban (recalcul seulement si filteredItems change)
- useCallback pour handlers (évite re-renders enfants)
- Filters debouncing côté client (recherche)
- Optimistic updates (UX instantané)

**Benchmarks estimés** :
- 50 cartes tâches : Drag fluide 60fps
- 100 opportunités : Recalcul colonnes < 10ms
- API call + rollback : < 300ms

**Scalabilité** :
- Si > 200 cartes par colonne : Virtualisation recommandée (react-window)
- Si > 1000 items total : Pagination serveur requise
